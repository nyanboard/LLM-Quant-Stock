# 数据管道审查

## 输入
`$ARGUMENTS` = 本次改动涉及的数据层文件或 change id

## 检查项
1. 是否存在前视偏差（使用了未来数据）
2. 数据缓存 TTL 是否合理（行情数据 vs 财报数据）
3. 股票代码格式是否统一（纯数字 vs 带前缀）
4. 是否处理了停牌、退市等特殊情况
5. 回测是否使用了复权数据
6. 外部 API 调用是否有重试和降级机制
7. 是否有数据有效性校验（空值、异常值）
8. 数据源切换是否会影响数据格式

## 输出格式
- 严重问题
- 警告问题
- 建议项
