# 隐性业务约定

## A 股数据相关约定

### 股票代码格式
- 背景：akshare 使用纯数字格式（如 `"000001"`），baostock 使用带前缀格式（如 `"sh.600000"`、`"sz.000001"`）
- 代码现状：统一在数据层做格式转换，上层模块一律使用纯数字格式
- 风险：格式混用会导致数据查询失败，且不报明显错误
- 处理要求：
  - 新增数据源适配时，必须在 DataSource 实现中统一转换为纯数字格式
  - 禁止在 Agent 或 Quant 层做格式转换

### 复权数据
- 背景：回测必须使用前复权（qfq）数据，实时分析可使用不复权数据
- 代码现状：数据层提供 `adjust` 参数，默认返回前复权数据
- 风险：使用不复权数据回测会导致收益率计算错误（分红、拆股影响）
- 处理要求：
  - 回测场景必须显式指定 `adjust="qfq"`
  - 实时行情使用不复权数据

### 停牌股票
- 背景：停牌股票在数据源中可能返回空数据或上一交易日数据
- 代码现状：未统一处理
- 风险：停牌股票进入选股名单会导致无法交易
- 处理要求：
  - 数据层必须标记停牌状态
  - 选股流程中必须过滤停牌股票

## Agent 输出约定

### 评分范围
- 背景：所有 Agent 的 score 统一为 1-10 分，confidence 为 0-1
- 代码现状：约定在 AgentSignal 数据类中
- 风险：评分范围不一致会导致综合评分失效
- 处理要求：
  - 新增 Agent 必须遵守 1-10 评分 + 0-1 信心度的格式
  - 综合评分模块依赖此格式

### 基金经理输出格式
- 背景：基金经理 Agent 输出的初选名单必须包含 symbol、score、reasoning 三个字段
- 代码现状：通过 FundManagerOutput 数据类约束
- 风险：缺失字段会导致量化层解析失败
- 处理要求：
  - 基金经理 Agent 的输出必须严格遵循数据类定义
  - 修改输出格式时必须在 design.md 中说明影响面

## 量化规则约定

### 综合评分权重
- 背景：LLM评分(40%) + 技术指标(30%) + 形态信号(20%) + 资金流(10%)
- 代码现状：权重配置在 `config/settings.py` 中
- 风险：修改权重会直接影响选股结果
- 处理要求：
  - 调整权重前必须跑回测对比
  - 不允许在代码中硬编码权重
