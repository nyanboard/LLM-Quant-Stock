# 架构总览

## 系统分层

```
┌─────────────────────────────────────────────┐
│  workflow/    编排层                          │
│  流程编排，串联各层，不写核心逻辑              │
├─────────────────────────────────────────────┤
│  agents/      Agent层（第一层：LLM选股）       │
│  多Agent协作，定性分析，输出初选名单            │
├─────────────────────────────────────────────┤
│  quant/       量化层（第二层：二次筛选）        │
│  技术指标、形态识别、综合评分                   │
├─────────────────────────────────────────────┤
│  data/        数据层                          │
│  统一数据接口，akshare/baostock 双数据源       │
├─────────────────────────────────────────────┤
│  backtest/    回测层                          │
│  策略回测、绩效评估                            │
├─────────────────────────────────────────────┤
│  config/      配置层                          │
│  全局配置、提示词模板、筛选规则                  │
└─────────────────────────────────────────────┘
```

## 依赖方向
- `workflow/` → `agents/` → `data/`
- `workflow/` → `quant/` → `data/`
- `workflow/` → `backtest/`
- `agents/` 不允许直接依赖 `quant/`
- `quant/` 不允许直接依赖 `agents/`
- 所有层通过 `data/` 获取数据，不允许绕过数据层直接调 API

## 模块职责

| 模块 | 职责 | 禁止 |
|------|------|------|
| `agents/` | LLM Agent 定义、提示词调用、信号输出 | 直接调 TA-Lib、直接访问外部 API |
| `quant/` | 技术指标计算、形态识别、评分过滤 | 调用 LLM、包含业务逻辑 |
| `data/` | 数据获取、缓存、清洗 | 包含选股逻辑 |
| `workflow/` | 流程编排 | 包含分析计算逻辑 |
| `backtest/` | 回测引擎、绩效报告 | 修改数据或策略逻辑 |
| `config/` | 配置管理、提示词模板 | 包含可执行逻辑 |

## 高风险区域
以下区域修改时必须重点说明：
- Agent 提示词模板（直接影响选股结果）
- 量化筛选规则（直接影响过滤结果）
- 数据获取接口（影响数据质量）
- 评分权重配置（影响最终排序）
- 回测引擎（影响绩效评估准确性）
