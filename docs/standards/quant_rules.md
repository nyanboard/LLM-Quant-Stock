# 量化策略与数据规范

## 基本规则
- 所有技术指标计算必须基于复权后的价格数据
- 不允许使用未来数据（前视偏差），所有指标只能使用当前及历史数据
- 筛选规则的参数必须可配置，不允许硬编码
- 新增技术指标必须说明适用场景和局限性
- 资金流向数据有时效性，缓存 TTL 不超过 1 天

## 技术指标计算要求
- RSI：默认周期 14，需处理边界情况（数据不足时返回 None）
- MACD：默认 fast=12, slow=26, signal=9
- 均线：支持 5/10/20/60/120/250 日均线
- BOLL：默认周期 20，标准差倍数 2.0
- KDJ：默认周期 9,3,3

## K 线形态识别
- 使用 TA-Lib 的 candlestick 函数
- 每个形态函数返回 0/1/-1（无信号/看涨/看跌）
- 形态信号必须与趋势方向结合判断，不单独作为交易信号

## 筛选规则配置
- 筛选规则通过 `config/quant_rules.yaml` 配置
- 规则分为三类：required（必须满足）、bonus（加分项）、veto（一票否决）
- 修改筛选规则前，必须用历史数据验证影响范围
- 不允许添加过于苛刻的条件导致无股票通过

## 回测规范
- 回测必须使用前复权数据
- 必须考虑交易成本（默认手续费 0.1%，印花税 0.1%）
- 必须考虑滑点（默认 0.1%）
- 持仓周期默认按周调仓
- 回测报告必须包含：累计收益、年化收益、最大回撤、夏普比率、胜率、vs 基准超额收益

## 必须说明的内容
以下场景必须在 design.md 或 review 摘要中说明：
- 新增技术指标的数学定义和适用条件
- 筛选规则变更对历史选股结果的影响
- 评分权重调整的回测对比数据
- 数据源变更可能引入的数据差异
