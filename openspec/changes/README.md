# OpenSpec Changes
> 当前正在执行的变更

本目录用于管理 OpenSpec 变更工件。

## 变更流程

1. `proposal.md` — 需求提案（做什么、为什么做）
2. `design.md` — 技术方案（怎么做、影响范围）
3. `tasks.md` — 执行步骤（拆解为可验证的任务节点）

## 当前变更

尚无活跃变更。使用以下命令创建新变更：

```
mkdir -p openspec/changes/<change-id>/specs
touch openspec/changes/<change-id>/proposal.md
touch openspec/changes/<change-id>/design.md
touch openspec/changes/<change-id>/tasks.md
```

## 归档

已完成的变更归档到 `archive/` 目录。
